# SHA-256 Hashing

#### Step-1 — No extra package required

SHA-256 is built-in to .NET (`System.Security.Cryptography`)

#### Step-2 — Hash Password During Registration
```c#
using System.Security.Cryptography;
using System.Text;

private static string HashPasswordSHA256(string password)
{
    using var sha256 = SHA256.Create();
    var bytes = Encoding.UTF8.GetBytes(password);
    var hash = sha256.ComputeHash(bytes);
    return Convert.ToHexString(hash); // Output HEX string
}

// during registration
var hash = HashPasswordSHA256(password);
user.PasswordHash = hash; // save to database
```

#### Step-3 — Verify During Login
```c#
var loginHash = HashPasswordSHA256(password);

if (loginHash != user.PasswordHash)
    return Unauthorized("Invalid credentials");
```

## Angular (Frontend)
Send username + password normally:

```ts
this.http.post('/api/sha256/login', {
  userName: this.userName,
  password: this.password
});
```
- No hashing on frontend
- Hashing is always server-side

**Important Notes**
- SHA-256 is a fast hashing algorithm
- Originally designed for data integrity, not passwords
- Easily brute-forceable on GPUs (billions per second)
- NOT recommended for password storage in real applications
- Better alternatives: BCrypt, Argon2, PBKDF2
